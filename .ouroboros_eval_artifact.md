# Pre-flight Readiness Checklist — FGF → PHASE2 AS2 Inbound Test

Four prerequisite domains: (1) network/firewall reachability, (2) TLS termination, (3) certificate format and exchange channel, (4) algorithm compatibility (SHA-256).

## 1. Network / firewall readiness
Container app must be Running with ≥1 healthy replica (deploy/azure-deploy.sh:83). Ingress external=true (azure-deploy.sh:82). No IP allowlist per task constraint. FGF egress must reach *.canadacentral.azurecontainerapps.io on TCP/443 (FGF-side).
Verify on live: curl -sS -o /dev/null -w "HTTP %{http_code}" "https://${FQDN}/as2" → expect HTTP 405 (POST-only handler at ServletConfig.java:75).

## 2. TLS requirements
TLS terminates at Container Apps front door. Public cert is Microsoft-managed for *.canadacentral.azurecontainerapps.io (FGF must NOT pin). TLS 1.2+ only. HTTP→HTTPS redirect via allowInsecure: false. No mTLS (clientCertificateMode: Ignore).
Verify: openssl s_client probe + curl http→https redirect probe. Expected MS-managed CA issuer, in-date cert, HTTP 301/308 redirect.

## 3. Certificate format and exchange channel
Accepted formats: DER (.cer/.der/.crt) — binary 0x30 0x82 — for Windows/IBM/Cleo/Mendelson; PEM (.pem/.crt) — Base64-wrapped — for OpenSSL/Linux. PHASE2.cer (DER 835 B) and PHASE2.pem (PEM 1208 B) both available.
NOT exchanged: .p12/.pfx (private key), .jks (deprecated JDK 17+), self-signed chain bundles.
Out-of-band channels: email ✓, secure file transfer ✓ recommended, in-person/phone hash dictation ✓ recommended for fingerprints, public URL ⚠ with fingerprint, Slack/Teams DM ⚠.
Fingerprint verification (both directions): openssl x509 -fingerprint -sha256 vs keytool -list -v. Authoritative PHASE2.cer SHA-256: AE:81:63:60:D4:D8:01:04:63:FF:F3:3E:6B:1E:F4:AA:F4:35:D2:05:1C:6B:3B:84:02:EB:8D:A1:F0:63:2A:74. Pre-flight requires written FGF confirmation of independent fingerprint match.
8-item cert exchange checklist: FGF cert delivered DER/PEM; imported under FGFCAAS21295T alias; certs.p12 uploaded to Azure Files config share; container restarted (or 300 s elapsed per config.xml:25); PHASE2.cer delivered to FGF; FGF recomputed PHASE2.cer fingerprint; FGF imported PHASE2.cer; PHASE2 recomputed FGFCAAS21295T.cer fingerprint.

## 4. Algorithm compatibility (SHA-256 signing + MIC)
FGF S/MIME signing: SHA-256 with RSA. MIC algo: lowercase `sha-256` (RFC 5751 + AS2 1.2 token format).
Disposition-Notification-Options header (canonical, copy-pasteable):
  signed-receipt-protocol=required,pkcs7-signature; signed-receipt-micalg=required,sha-256
Required values force signed MDN with SHA-256 MIC, overriding SelfFillingPartnershipFactory SHA-1 default at SelfFillingPartnershipFactory.java:78-79.
Encryption out of scope but if FGF encrypts: AES-128/192/256-CBC, AES-128/256-GCM all decrypt cleanly. Compression optional.
Keystore requirements: alias PHASE2 RSA-2048 SHA256withRSA; alias fgfcaas21295t trustedCertEntry; mounted at /app/config/certs.p12.
SelfFillingPartnershipFactory failure mode: if FGF's first message omits signed-receipt-micalg or sends 'optional', partnership auto-creates with SHA-1 default and MIC mismatch occurs. Recovery: delete /app/data/partnerships.xml + restart.
7-item algorithm checklist: FGF AS2 1.2 + SHA-256 + signed MDN confirmed; FGF will send exact Disposition-Notification-Options; PHASE2 keystore alias signature-algo verified; no stale partnerships.xml; restart applied if deleted.
